# KUBERNETES OPERATOR

## OVERVIEW
Go operator managing the `GammaInstance` CRD. Declaratively provisions all infra for a Gamma deployment: game server, client, Redis, Ingress, and HPA.

## KEY FILES
| File | Purpose |
|------|---------|
| `api/v1alpha1/gammainstance_types.go` | CRD spec/status types |
| `internal/controller/gammainstance_controller.go` | Main reconciler |
| `internal/controller/server.go` | Server Deployment + Service |
| `internal/controller/client.go` | Client Deployment + Service |
| `internal/controller/redis.go` | Redis StatefulSet + ConfigMap + Service |
| `internal/controller/networking.go` | Ingress (WebSocket-aware nginx) |
| `internal/controller/autoscaling.go` | HPA for server |
| `cmd/main.go` | Operator entrypoint |
| `api/v1alpha1/zz_generated.deepcopy.go` | **DO NOT EDIT** — generated by `controller-gen` |

## RECONCILE ORDER
`Redis → Server → Client → Ingress → HPA → update CR status`

Status phases: `Deploying` → `Running` | `Degraded` | `Failed`

## CRITICAL IMPLEMENTATION DETAILS
- **Sticky sessions**: Server Service uses `SessionAffinity: ClientIP` (3600s timeout) — required because Colyseus rooms are in-memory. Don't remove this.
- **WebSocket ingress**: `/ws` → server port, `/` → client port. Nginx annotations for proxy-http-version 1.1 and connection upgrade are auto-added.
- **Client server URL**: if ingress is enabled, operator derives and injects `GAMMA_SERVER_URL` into the client container automatically.
- **Redis env injection**: when `redis.enabled: true`, operator sets `REDIS_URL` and `STATE_BACKEND=redis` on the server deployment.

## ANTI-PATTERNS
- Never edit `zz_generated.deepcopy.go` — run `make generate` instead.
- Do not remove `SessionAffinity` from the server Service — WebSocket clients will disconnect on pod changes.
- Do not add game logic to the operator — it only manages infrastructure.
